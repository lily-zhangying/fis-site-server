示例安装

我们提供一个demo让用户下载安装，直观体验FIS的开发模式.

  • 首先安装FIS本地的调试预览环境
$ fisp server init
  • 使用lights下载demo到本地
$ lights install pc-demo
  • ok，demo已经下载好啦，我们在本地来看看demo效果。

示例预览

demo中包括common和home两个模块，目录结构是这样的：

├── common   //公用资源模块
│   ├── fis-conf.js  //配置文件
│   ├── page     //页面模板目录
│   ├── plugin   //smarty plugin目录
│   ├── static   //静态资源目录
│   └── widget   //模块化组件目录
└── home    //业务模块
    ├── fis-conf.js
    ├── page
    ├── static
    ├── test    //测试数据目录
    └── widget

FIS的目录规范

  • 让代码保持简洁清晰
  • 划分模块的思路保证项目的并行开发，简单上线。
  • 极大提高了项目可控性，和工程师开发效率。
  • 具体规范可查看FIS的目录规范。

接下来，我们就可以在本地看demo效果啦。

我们发布common，home模块，启动FIS本地调试服务器，在浏览器中预览页面：

//进入common模块路径
$ cd common

//编译发布common模块
$ fisp release -c

//进入home模块路径
$ cd ../home

//编译发布home模块
$ fisp release -c

//启动本地server服务
$ fisp server start

//在浏览器中访问页面，看demo效果
http://localhost:8080/home/page/index

现在你就看到下面这样一个demo页面啦~

你可以尝试修改页面内容，重新发布，就能立刻看到自己修改的效果了。
